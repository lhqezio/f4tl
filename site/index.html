10 Pre-Built QA Workflows

MCP prompts that kick off complete testing scenarios. Invoke them from your MCP client.

smoke-test

Quick health check — navigate, screenshot, check for console errors and 4xx/5xx.

full-qa

Comprehensive pass — all pages, forms, a11y, responsive, generate HTML report.

regression-test

After a deploy — re-run key user flows, compare against a baseline session.

accessibility-audit

Audit ARIA labels, alt text, heading structure, keyboard navigation, contrast.

form-test

Test form validation — empty, valid, invalid, XSS payloads, boundary values.

performance-check

Check load times, large assets, slow API calls, excessive DOM nodes.

visual-inventory

Screenshot every page at desktop, tablet, and mobile viewports.

multi-actor-test

Coordinated multi-user test — buyer + seller + admin interacting together.

webhook-test

Discover endpoints, fire synthetic events, verify UI/state, test idempotency.

regression-run

Use session history to find gaps and regressions, then run targeted tests.

Get Started in 3 Steps

Works with any MCP client: Claude Desktop, Claude Code, Cursor, etc.

1

Install & generate config

npm install -g f4tl
cd your-project
f4tl init
2

Start the MCP server

f4tl serve
3

Configure your MCP client

{
  "mcpServers": {
    "f4tl": {
      "command": "f4tl",
      "args": ["serve"],
      "cwd": "/path/to/your/project"
    }
  }
}

Or start with the live dashboard:

f4tl serve --dashboard
